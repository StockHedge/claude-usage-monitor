# 사용설명서 — Claude 5시간 사용량 모니터

처음 쓰는 사람도 따라 할 수 있게 정리한 안내서입니다.

## 1. Python 설치 (한 번만)

1. https://www.python.org/downloads/ 에서 Windows용 Python 3.8 이상을 받습니다.
2. 설치 화면 맨 아래 **“Add python.exe to PATH”** 를 반드시 체크하고 설치합니다.
3. 이미 설치돼 있으면 건너뜁니다. (python.org 설치본에는 이 프로그램에 필요한 tkinter가 포함돼 있습니다.)

## 2. 프로그램 받기

- 이 저장소를 **Code → Download ZIP** 으로 받아 압축을 풀거나, `git clone` 합니다.
- 압축을 푼 폴더는 나중에 옮기지 말고 **그대로 두세요**(자동 시작이 이 위치를 참조합니다).

## 3. 최초 실행

1. 폴더 안의 **`install.bat` 을 더블클릭**합니다.
2. 뜨는 설정 창에서 **본인의 요금제**(Pro / Max 5x / Max 20x)를 고릅니다.
3. **“Windows 로그인 시 자동 시작”** 은 켜 두는 것을 권장합니다.
4. **“설치하고 시작”** 을 누르면 화면 **우상단에 작은 팝업**이 뜹니다.

> Claude Code에 로그인돼 있으면 `/usage`와 똑같은 **정확값**이 표시됩니다.
> 로그인 정보가 없으면 `~` 가 붙은 **추정값**으로 동작하다가, 로그인하면 자동으로 정확값으로 바뀝니다.

## 4. 평소 사용법

- **옮기기**: 팝업을 마우스로 드래그하면 원하는 위치로 이동합니다(위치는 자동 저장).
- **메뉴**: 팝업에서 **마우스 우클릭** → 지금 새로고침 / 설정… / 위치 초기화 / 종료.
- **리셋 시각 보기**: 팝업에 **마우스를 올리면** 5시간 한도가 몇 시에 리셋되는지, 얼마나 남았는지 알려줍니다.
- **음성 알림**: 사용량이 25% 구간을 넘길 때마다 "N퍼센트 소진했습니다"라고 한국어로 읽어줍니다(설정에서 끌 수 있음).
- **숫자 색**: 여유(정상) → 주의 → 위험 순으로 색이 바뀝니다. 기본은 초록/주황/빨강.

## 5. 모양·동작 바꾸기 (설정 창)

우클릭 → **설정…** 에서 아래를 바로 바꿀 수 있고, 저장하면 즉시 반영됩니다.

- 요금제, 캐릭터 아이콘 표시 여부, 음성 알림 켜기/끄기
- 숫자 크기 / 아이콘 크기 / 투명도
- 주의·위험 임계값(%) / 갱신 주기(초)
- 색상: 배경 · 정상 · 주의 · 위험 · 아이콘

## 6. 끄기 / 제거

- **잠깐 끄기**: 우클릭 → 종료. (다음 로그인 때 자동 시작으로 다시 뜹니다.)
- **자동 시작 해제**: **`uninstall.bat`** 더블클릭.
- **완전 제거**: 자동 시작 해제 후 프로그램 폴더를 삭제하고,
  설정/로그 폴더 `C:\Users\<사용자>\.claude_usage_monitor` 도 지우면 됩니다.

## 7. 문제 해결

- **팝업이 안 보여요**: `start.bat` 을 더블클릭해 수동 실행해 보세요. 그래도 안 되면
  `C:\Users\<사용자>\.claude_usage_monitor\monitor.log` 를 확인하세요.
- **숫자가 `--` 로 떠요**: 데이터를 아직 못 읽은 상태입니다. 잠시 기다리거나 우클릭 → 지금 새로고침.
- **계속 `~` (추정값)만 나와요**: Claude Code 로그인이 안 돼 있거나 토큰이 만료됐을 수 있습니다.
  Claude Code를 한 번 실행/로그인하면 정확값으로 바뀝니다.
- **`install.bat` 이 “Python not found” 라고 해요**: 1번(Python 설치, PATH 체크)을 다시 확인하세요.
- **한글이 깨져 보여요**: 이 폴더의 파일을 **UTF-8** 로 유지하세요. 배치 파일은 그대로 두면 됩니다.

## 8. 개인정보·보안

이 프로그램은 사용량 숫자를 읽기 위해 `~/.claude/.credentials.json` 의 토큰을 **읽기만** 합니다.
토큰을 저장·전송·변경하지 않으며, 모든 처리는 본인 PC 안에서만 이뤄집니다.
